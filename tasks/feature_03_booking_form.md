# Feature 03. Форма записи на кулинарный класс

## Цель задачи

Реализовать форму записи клиента на выбранный кулинарный класс.

Пользователь должен иметь возможность открыть подробную карточку класса, нажать кнопку «Записаться», заполнить контактные данные, выбрать вариант по фартуку и набору ножей, указать количество участников и получить подтверждение записи с итоговой стоимостью.

---

## Связанные требования

- FR-010 — приложение должно отображать форму записи на выбранный класс.
- FR-011 — форма должна содержать обязательные поля для имени и телефона клиента.
- FR-012 — приложение должно позволять указать количество участников.
- FR-016 — клиент должен иметь возможность указать аллергии или пищевые ограничения.
- FR-019 — клиент должен иметь возможность выбрать вариант по фартуку и набору ножей.
- FR-020 — приложение должно проверять доступность комплектов студии.
- FR-022 — приложение не должно создавать запись, если свободных мест нет.
- FR-025 — после успешной записи приложение должно показывать статус подтверждения.
- FR-026 — после записи приложение должно обновлять количество свободных мест.

---

## Что было реализовано

- Добавлен контейнер `bookingView` в `index.html`.
- В mock-данные добавлено временное хранилище записей `mockBookings`.
- В mock API добавлен метод `createBooking(payload)`.
- В подробную карточку класса добавлена кнопка «Записаться».
- Реализована форма записи с полями:
    - имя;
    - телефон;
    - email;
    - количество участников;
    - аллергии или пищевые ограничения;
    - выбор фартука и набора ножей;
    - комментарий.
- Добавлена проверка обязательных полей.
- Добавлена проверка корректности номера телефона.
- Телефон после успешной записи отображается в едином формате.
- Добавлена проверка свободных мест.
- Добавлена проверка доступности комплектов студии.
- После успешной записи количество свободных мест уменьшается.
- После успешной записи количество доступных комплектов студии уменьшается, если клиент выбрал комплект студии.
- Добавлен расчёт стоимости занятия.
- Добавлен расчёт стоимости комплекта студии.
- Добавлен расчёт итоговой стоимости записи.
- После успешной записи отображается экран подтверждения с данными клиента, количеством участников, статусом и итоговой стоимостью.
- Добавлена кнопка возврата к списку после успешной записи.

## Использованные промпты

```text
Реализуй третью фичу для клиентского приложения кулинарной студии «Шеф-стол»: форму записи на выбранный класс.

Нужно:
- добавить кнопку «Записаться» в подробную карточку класса;
- по клику открывать форму записи;
- форма должна содержать имя, телефон, email, количество участников, аллергии, выбор фартука и набора ножей, комментарий;
- обязательными должны быть имя, телефон и количество участников;
- данные должны отправляться через mock API методом createBooking(payload);
- если свободных мест нет, запись не должна создаваться;
- если клиент выбрал комплект студии, нужно проверить количество доступных комплектов;
- после успешной записи показать подтверждение;
- после записи уменьшить количество свободных мест;
- сохранить разделение кода по файлам: mock-data.js, api.js, ui.js, app.js.
```
## Ручная проверка

Проверка выполнялась через открытие файла `src/index.html` в браузере.

Проверено:

- список кулинарных классов загружается без ошибок;
- кнопка «Подробнее» открывает подробную карточку выбранного класса;
- в подробной карточке отображается кнопка «Записаться»;
- кнопка «Записаться» открывает форму записи на выбранный класс;
- в форме отображаются поля для имени, телефона, email, количества участников, аллергий, выбора фартука и набора ножей, комментария;
- форма не отправляется, если поле имени не заполнено;
- форма не отправляется, если поле телефона не заполнено;
- форма не отправляется, если телефон введён неполностью;
- корректный телефон после успешной записи отображается в едином формате;
- email проверяется средствами браузера;
- при корректно заполненных данных запись создаётся;
- после успешной записи отображается экран «Вы записаны!»;
- на экране успешной записи отображаются имя, телефон, количество участников и статус записи;
- на экране успешной записи отображается итоговая стоимость;
- если выбран комплект студии, стоимость комплекта добавляется к итоговой сумме;
- после успешной записи количество свободных мест уменьшается;
- если выбран комплект студии, количество доступных комплектов студии уменьшается;
- если свободных мест недостаточно, запись не создаётся и отображается ошибка;
- если комплектов студии недостаточно, запись не создаётся и отображается ошибка;
- кнопка «Вернуться к списку» возвращает пользователя к списку кулинарных классов;
- интерфейс формы корректно отображается в мобильном размере браузера.

Результат проверки: форма записи работает корректно, данные проходят валидацию, запись создаётся через mock API, а пользователь получает подтверждение с итоговой стоимостью.
