# User Stories — «Кулинарная студия»

## Назначение документа

Документ фиксирует пользовательские истории для клиентского мобильного приложения кулинарной студии «Шеф-стол».

User Story описывает потребность пользователя в формате:

> Как `<роль>`, я хочу `<действие>`, чтобы `<ценность>`.

Истории в этом документе описывают только роль клиента. Интерфейсы владельца, администратора и шефа находятся вне текущей поставки.

---

## 1. Контекст и границы

В рамках текущей поставки разрабатывается клиентское мобильное приложение и API-контракт для него.

Клиентское приложение должно позволить клиенту:

- просматривать доступные кулинарные классы;
- открывать подробную информацию о классе;
- записываться на выбранный класс;
- передавать данные, необходимые для подготовки занятия;
- видеть статус своей записи;
- корректно получать информацию об отмене класса студией.

Клиентское приложение не создаёт и не редактирует расписание, программы классов и данные о шефах. Эти данные поступают из существующего бэкенда через API.

---

## 2. Приоритеты

| Приоритет | Значение |
|---|---|
| Высокий | История нужна для базового сценария MVP. Без неё клиент не сможет полноценно выбрать класс и записаться. |
| Средний | История важна для удобства и прозрачности, но может быть реализована после базового сценария. |
| Низкий | История полезна для развития продукта, но не является обязательной для первой версии. |

## 3. User Stories для MVP

### US-001. Просмотр доступных кулинарных классов

| Поле | Значение |
|---|---|
| ID | US-001 |
| Приоритет | Высокий |
| Связанные бизнес-требования | BR-001, BR-003, BR-004 |
| Статус | MVP |

**User Story**

Как клиент, я хочу видеть список доступных кулинарных классов, чтобы выбрать подходящее занятие по дате, времени и программе.

**Критерии приёмки**

| ID | Дано | Когда | Тогда |
|---|---|---|---|
| AC-001.1 | В системе есть доступные классы | Клиент открывает приложение | Клиент видит список доступных классов |
| AC-001.2 | Класс доступен для записи | Клиент просматривает список | В карточке отображаются дата, время, программа, шеф и количество свободных мест |
| AC-001.3 | На ближайшие дни нет доступных классов | Клиент открывает список | Клиент видит понятное сообщение, что доступных классов пока нет |

## 7. Итог

Для MVP ключевыми являются истории, которые закрывают базовый путь клиента:

1. увидеть доступные кулинарные классы;
2. открыть подробную информацию о классе;
3. записаться на выбранный класс;
4. передать важные данные для подготовки занятия;
5. получить понятный статус записи;
6. увидеть ошибку, если запись невозможна;
7. увидеть статус отмены, если класс отменён студией.

Истории про онлайн-оплату, программу для постоянных клиентов и расширенные уведомления оставлены после MVP, потому что они увеличивают объём проекта и не являются обязательными для решения основной проблемы ручной записи.
