# Функциональные требования — «Кулинарная студия»

## Назначение документа

Документ фиксирует функциональные требования к клиентскому мобильному приложению кулинарной студии «Шеф-стол».

Функциональные требования описывают, что должна делать система в рамках текущей поставки.  
Документ не описывает внутреннюю реализацию бэкенда, админку, интерфейс владельца или интерфейс шефа.

---

## 1. Границы функциональных требований

В текущую поставку входит:

- просмотр списка доступных кулинарных классов;
- просмотр подробной карточки класса;
- запись клиента на выбранный класс;
- ввод контактных данных;
- передача информации об аллергиях;
- выбор по фартуку и набору ножей, если эта опция подтверждена как часть MVP;
- отображение статуса записи;
- отмена записи клиентом, если правило отмены подтверждено;
- отображение отмены класса студией;
- обработка ошибок API и пустых состояний.

В текущую поставку не входит:

- создание и редактирование расписания;
- управление программами классов;
- назначение шефов;
- интерфейс администратора;
- интерфейс владельца;
- интерфейс шефа;
- внутренняя логика бэкенда;
- онлайн-оплата как обязательная часть MVP;
- программа лояльности;
- управление закупками продуктов.

## 15. Итог

Функциональные требования MVP сосредоточены на основном пользовательском пути:

1. клиент открывает список доступных кулинарных классов;
2. выбирает подходящий класс;
3. открывает подробную карточку;
4. заполняет форму записи;
5. отправляет запись;
6. получает подтверждение или понятную ошибку;
7. видит актуальный статус записи.

Функции, которые не являются необходимыми для базового сценария записи, вынесены после MVP или требуют дополнительного подтверждения заказчиком.
