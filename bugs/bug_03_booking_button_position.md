# Bug 03. Кнопка «Записаться» отображалась не в той части интерфейса

## Симптом

После добавления формы записи кнопка «Записаться» сначала была добавлена не в подробную карточку класса, а в карточку списка.

Из-за этого пользователь мог видеть кнопку записи слишком рано, до просмотра подробной информации о классе.

## Связанное требование

- FR-006 — приложение должно отображать подробную информацию о выбранном классе.
- FR-010 — форма записи должна открываться из карточки выбранного класса.
- US-003 — как клиент, я хочу записаться на выбранный кулинарный класс после просмотра информации о нём.

## Как воспроизвести

1. Открыть `src/index.html` в браузере.
2. Посмотреть список кулинарных классов.
3. Проверить кнопки внутри карточек списка.

## Фактический результат

Кнопка «Записаться» могла отображаться в карточке списка вместе с кнопкой «Подробнее».

## Ожидаемый результат

В списке классов должна отображаться только кнопка «Подробнее».  
Кнопка «Записаться» должна быть доступна в подробной карточке класса, после того как пользователь посмотрел описание занятия, меню, шефа, цену, адрес и информацию о комплектах студии.

## Причина

HTML-блок кнопки записи был добавлен в функцию отрисовки карточки списка `renderClassCard(slot)`, а не в функцию подробной карточки `renderClassDetails(slot)`.

## Исправление

Кнопка «Записаться» была удалена из карточки списка и перенесена в подробную карточку класса.

Теперь:

- в списке отображается кнопка «Подробнее»;
- в подробной карточке отображается кнопка «Записаться»;
- форма записи открывается только после перехода в подробную карточку.

## Использованный промпт

```text
Проверь расположение кнопки «Записаться». Она должна быть не в карточке списка, а в подробной карточке выбранного класса. Список должен вести к просмотру подробностей, а запись должна начинаться уже после просмотра карточки.
```

## Ручная проверка

Проверка выполнялась через открытие файла `src/index.html` в браузере.

Проверено:

- в списке классов отображается кнопка «Подробнее»;
- кнопка «Записаться» не отображается в карточках списка;
- при нажатии «Подробнее» открывается подробная карточка;
- в подробной карточке отображается кнопка «Записаться»;
- при нажатии «Записаться» открывается форма записи;
- пользовательский сценарий стал логичным: список → подробная карточка → форма записи.

## Результат

Баг исправлен.  
Кнопка «Записаться» находится в правильном месте пользовательского сценария.
