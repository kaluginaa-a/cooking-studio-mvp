# Bug 01. Форма принимала неполный номер телефона

## Симптом

Во время ручной проверки формы записи было обнаружено, что поле телефона принимало неполный номер, например `8962`.

Форма позволяла создать запись, хотя номер телефона был введён некорректно.

## Связанное требование

- FR-011 — форма должна содержать обязательные поля для имени и телефона клиента.
- FR-025 — после успешной записи приложение должно показывать подтверждение записи только при корректно заполненных данных.

## Как воспроизвести

1. Открыть `src/index.html` в браузере.
2. Открыть подробную карточку доступного класса.
3. Нажать «Записаться».
4. Ввести имя.
5. Ввести в поле телефона значение `8962`.
6. Нажать «Подтвердить запись».

## Фактический результат

Форма принимала неполный номер телефона и создавала запись.

## Ожидаемый результат

Форма не должна создавать запись, если телефон введён неполностью.  
Пользователь должен увидеть сообщение об ошибке.

## Причина

Поле телефона имело тип `tel`, но сам по себе `type="tel"` не проверяет корректность номера.  
В коде была проверка только на пустое значение, но не было проверки длины и формата телефона.

## Исправление

В `src/js/app.js` были добавлены функции:

- `getPhoneDigits(phone)` — оставляет только цифры из введённого номера;
- `isValidPhone(phone)` — проверяет, что номер содержит 10 цифр или 11 цифр с началом на `7` или `8`;
- `formatPhone(phone)` — приводит номер к единому виду.

Также была обновлена функция `validateBookingPayload(payload)`.  
Теперь форма не отправляется, если телефон введён неполностью.

## Использованный промпт

```text
Добавь проверку телефона в форме записи. Сейчас email проверяется браузером, а телефон можно ввести неполностью. Нужно запретить отправку формы, если телефон введён некорректно, и после успешной записи показывать телефон в едином формате.
```

## Ручная проверка

Проверка выполнялась через открытие файла `src/index.html` в браузере.

Проверено:

- значение `8962` не проходит проверку;
- при неполном телефоне запись не создаётся;
- отображается сообщение «Введите телефон полностью, например +7 900 000-00-00.»;
- номер `+7 900 123-45-67` проходит проверку;
- номер `89001234567` проходит проверку;
- после успешной записи телефон отображается в едином формате.

## Результат

Баг исправлен.  
Форма записи больше не принимает неполные номера телефона.
