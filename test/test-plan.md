# Ручной тест-план MVP «Шеф-стол»

## 1. Цель тестирования

Цель ручного тестирования — проверить основные пользовательские сценарии клиентского MVP кулинарной студии «Шеф-стол»:

- просмотр списка ближайших кулинарных классов;
- открытие подробной карточки класса;
- запись клиента на выбранный класс;
- проверка обязательных полей формы;
- проверка телефона и email;
- расчёт итоговой стоимости;
- обновление количества свободных мест и комплектов студии;
- обработка ошибок и недоступных состояний.

Тестирование выполняется вручную через браузер, без подключения реального backend.  
Данные берутся из mock API и файла `src/js/mock-data.js`.

---

## 2. Тестируемая область

| Раздел | Что проверяется |
|---|---|
| Список классов | Загрузка карточек, отображение основной информации, статусы мест |
| Карточка класса | Подробная информация о занятии, шефе, меню, адресе и комплектах студии |
| Форма записи | Ввод данных клиента, выбор количества участников, выбор комплекта |
| Валидация | Обязательные поля, телефон, email, количество участников |
| Mock API | Создание записи, проверка мест, проверка комплектов студии |
| Подтверждение записи | Отображение данных клиента, статуса и итоговой стоимости |

## 3. Тестовое окружение

| Параметр | Значение |
|---|---|
| Тип тестирования | Ручное функциональное тестирование |

```text
Сгенерируй ручные тест-кейсы для клиентского MVP кулинарной студии «Шеф-стол».

Контекст проекта:
- приложение показывает список ближайших кулинарных мастер-классов;
- пользователь может открыть подробную карточку класса;

- форма содержит имя, телефон, email, количество участников, аллергии, выбор фартука и набора ножей, комментарий;
```
